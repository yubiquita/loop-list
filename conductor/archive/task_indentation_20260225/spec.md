# Specification: Task Grouping by Indentation (タスクのインデントによるグループ化機能)

## 概要
タスクを一段階インデントして親タスクの下にグループ化できるようにします。これにより、チェックリスト内の関連する項目を整理し、管理しやすくします。

## ユーザーストーリー
- ユーザーとして、タスクを右にスワイプして、上のタスクの子要素としてインデントしたい。
- ユーザーとして、インデントされたタスクを左にスワイプして、インデントを解除したい。
- ユーザーとして、親タスクにチェックを入れたとき、その子タスクも自動的にチェックされるようにしたい。
- ユーザーとして、親タスクのチェックを外したとき、その子タスクのチェックもすべて自動的に外れるようにしたい。

## 機能要件
### 1. インデント操作
- タスクは最大1段階（親->子）までインデント可能。
- 右スワイプでインデント、左スワイプでインデント解除（アウトデント）。
- モバイルデバイスでの直感的なスワイプ操作に対応。

### 2. 視覚的表現
- インデントされたタスクは、シンプルな左マージンを設けて表示。
- ツリー線などの複雑な装飾は行わず、ミニマルなデザインを維持。

### 3. 階層構造の挙動
- **カスケードチェック/クリア:** 親タスクがチェックされた際、その直下に続くすべてのインデントされた子タスクも自動的にチェック状態にする。逆に、親タスクのチェックが外された（未チェックに戻された）際も、紐付くすべての子タスクを未チェック状態にする。
- **データの永続化:** インデント状態（階層情報）は LocalStorage に保存され、再読み込み後も維持される。

## 非機能要件
- **モバイルパフォーマンス:** スワイプジェスチャーはスムーズで、入力遅延を感じさせないこと。
- **アクセシビリティ:** スワイプ操作以外に（必要に応じて）代替手段や視覚的なフィードバックを提供。

## 受け入れ条件
- [ ] タスクを右スワイプすると、適切にマージンが付与されてインデントされること。
- [ ] インデントされたタスクを左スワイプすると、マージンが消えて元の位置に戻ること。
- [ ] 親タスクをチェックした際、紐付く子タスクがすべてチェックされること。
- [ ] 親タスクのチェックを外した際、紐付く子タスクがすべて未チェック状態になること。
- [ ] ページをリロードしてもインデント状態が保持されていること。
- [ ] インデントは1段階のみで、それ以上の深い階層は作成できないこと。

## スコープ外
- 2段階以上の深い階層構造。
- グループの折りたたみ（エクスパンド/コラプス）機能。
- ドラッグ＆ドロップによる階層の入れ替え。
