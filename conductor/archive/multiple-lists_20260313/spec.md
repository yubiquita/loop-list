# 仕様書: 複数のルーチンリスト対応

## 概要
Loop List アプリケーション内で複数のルーチンリストを作成・管理できるようにします。ユーザーはドロップダウンメニューを使用してリストを切り替え、新しいリストの作成、名前の変更、および削除を行うことができます。最後に使用したリストは保存され、アプリの再起動時に復元されます。

## 機能要件

### 1. リスト管理 (CRUD)
- **作成:** ユーザーはデフォルト名（例：「新しいリスト」）で新しい空のリストを追加できます。
- **閲覧:** ユーザーは選択したリストの内容を表示できます。
- **更新 (名前変更):** ユーザーは既存のリストの名前を編集できます。
- **削除:** ユーザーはリストを削除できます。アクティブなリストが削除された場合、アプリは別の利用可能なリストに切り替えるか、デフォルトのリストを作成します。
- **並び替え:** ユーザーはドロップダウン内のリストの表示順序を変更できます。

### 2. リストの切り替え
- **ドロップダウンメニュー:** 画面上部の UI に現在のリスト名を表示し、他のリストへの切り替えを可能にします。
- **アクティブ状態:** アプリケーションは現在どのリストがアクティブであるかを追跡し、そのタスクを表示します。

### 3. データの永続化
- **LocalStorage:** すべてのリストとそのタスクは、LocalStorage 内に単一の JSON オブジェクトとして保存されます。
- **セッションの復元:** 最後にアクティブだったリストの ID を保存し、アプリ起動時にそのリストをロードします。
- **初期移行:** ユーザーの要望に基づき、既存の単一リストデータはリセットし、新しい複数リスト構造への移行を簡素化します。

## ユーザーインターフェース (UI)
- **ヘッダードロップダウン:** 現在のヘッダーを拡張し、アクティブなリスト名を表示・選択できるようにします。
- **管理画面:** リストの名前変更、削除、並び替え操作を行うためのシンプルなインターフェース（ドロップダウン内または設定アイコンからアクセス）。

## 受付基準 (Acceptance Criteria)
- [ ] ユーザーが新しいリストを作成でき、それがドロップダウンに表示されること。
- [ ] ユーザーが複数のリストを切り替えることができ、タスク一覧が正しく更新されること。
- [ ] ユーザーがリストの名前を変更でき、UI と LocalStorage に反映されること。
- [ ] ユーザーがリストを削除でき、アクティブなリストの削除も適切に処理されること。
- [ ] 並び替えたリストの順序が保持されること。
- [ ] アプリの再読み込み時に、最後に開いていたリストが復元されること。

## スコープ外
- クラウド同期。
- 個別のリストの書き出し/読み込み（将来の検討事項）。
- 他のユーザーとのリスト共有。
