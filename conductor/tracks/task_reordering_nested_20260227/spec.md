# Specification: Task Reordering (Nested Structure)

## 1. Overview
`loop-list` の「リスト詳細画面（Detail Screen）」において、ユーザーがタスクの順序を自由に入れ替えられる機能を導入します。モバイルでの直感的な操作を重視し、右側のハンドルを用いたドラッグ＆ドロップ（D&D）操作を採用します。また、階層構造（親子関係）を堅牢に維持するため、内部データ構造をフラットからネスト（再帰的）にリファクタリングします。

## 2. Functional Requirements
- **リスト詳細画面での常時有効化:** 
  - 並べ替えハンドルを右側に配置し、チェック操作やスワイプ操作（インデント）と共存させます。
- **データ構造のネスト化:**
  - `ChecklistItem` が `subItems: ChecklistItem[]` を持つ再帰的な構造に変更します。
  - 親タスクをドラッグすると、その子タスク（`subItems`）も一塊として移動します。
- **同一階層内での並べ替え:**
  - タスクは同じ親を持つ（＝同じ階層レベルの）タスク間でのみ順位を入れ替えることができます。
- **自動保存:**
  - 並べ替えが完了（ドロップ）した時点で、LocalStorage に新しい順序が自動的に保存されます。
- **データ移行 (Migration):**
  - 既存のフラットなデータ構造（`indent: boolean`）を、ネスト構造へ自動的に変換する処理を実装します。

## 3. Non-Functional Requirements
- **操作の分離:** 
  - 右側のハンドル：上下ドラッグ（並べ替え）
  - 中央領域：タップ（チェック）、左右スワイプ（インデント/アウトデント）
- **パフォーマンス:** `vuedraggable` を使用し、滑らかなアニメーションとモバイル対応を実現します。
- **堅牢性:** データ構造の変更に伴い、ストア（Pinia）のロジックとテストを全面的に更新します。

## 4. Acceptance Criteria
- [ ] 各タスクの右側にドラッグハンドルが表示されること。
- [ ] ハンドルを掴んで上下に動かし、同じ階層内でタスクを並べ替えられること。
- [ ] 親タスクを移動させると、その子タスクも一緒に移動すること。
- [ ] 並べ替え後、再読み込みしても順序が保持されていること。
- [ ] 既存のインデント操作（スワイプ）が正しく動作し、ネスト構造に反映されること。
- [ ] 既存のフラットなデータが、初回起動時に正しくネスト構造に変換されること。

## 5. Out of Scope
- D&Dによる階層（インデントレベル）の直接的な変更（引き続きスワイプ操作で行う）。
- 3段階以上の深い階層構造（現時点では1段階のインデントを維持）。
- PCブラウザでの複雑なショートカットキーによる並べ替え。
