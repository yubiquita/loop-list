# Specification - redesign_from_mock_20260313

## Overview
現在の `loop-list` プロジェクトの実装を大幅に見直し、提供された `ref/loop-list-mock.jsx` の UX/UI を Vue 3 と Vite を用いて再実装する。特に、スワイプ操作によるインデントの変更や、ドラッグ＆ドロップによるグループ単位の並び替え、チェックボックスのカスケード動作を忠実に再現する。

## Functional Requirements
- **タスク管理:**
  - タスクの追加、削除、チェック状態の切り替え。
  - 親タスクのチェック状態は、その配下の子タスク（`indent: 1`）すべてに連動する。
  - すべての子タスクがチェックされた場合、親タスクも自動的にチェックされる。
- **階層構造 (フラットデータ構造):**
  - タスクは `id`, `text`, `completed`, `indent` を持つフラットな配列で管理する。
  - `indent: 0` は親タスク、`indent: 1` は子タスクとする。
- **UX/UI 動作 (モックの忠実な再現):**
  - **スワイプによるインデント操作:**
    - 右スワイプで子タスク化 (`indent: 1`)。
    - 左スワイプで親タスク化 (`indent: 0`)。
    - スワイプ中に背景色が変わり、操作内容（「サブタスクにする」「親タスクにする」）を表示する。
  - **ドラッグ＆ドロップによる並び替え:**
    - 親タスクをドラッグすると、その配下の子タスク（グループ）ごと移動する。
    - 子タスク単体での移動も可能。
    - モックと同様のポインターイベントベースの自作ソートロジック（`performSort`）を移植する。
- **進捗表示:**
  - 全タスクに対する完了タスクの割合を表示する `ProgressBar` コンポーネントの維持・統合。

## Non-Functional Requirements
- **モバイルファースト:** タッチ操作（スワイプ、ドラッグ）のレスポンスと感度を最優先する。
- **Vue 3 / Vite:** 現在の技術スタックを維持しつつ、ロジックをモックから移植する。
- **テスト主導:** 重要なロジック（ソート、チェック連動）についてはテストを作成し、品質を担保する。
- **簡潔な状態管理:** 初期段階では Pinia を使用せず、`App.vue` 等での `ref` 管理、または単純な `reactive` オブジェクトでの管理とする。

## Acceptance Criteria
- モックで実現されているスワイプによるインデント変更が Vue コンポーネントでスムーズに動作すること。
- 親タスクをドラッグした際に、その配下の子タスクがひとまとまりとして移動すること。
- チェックボックスの連動（カスケードチェック）が正しく動作すること。
- 既存の `ProgressBar` が、新しいフラットなデータ構造に基づいて正しく進捗を表示すること。

## Out of Scope
- LocalStorage への自動保存機能（本トラックではメモリ内での動作を確認し、保存は後回しとする）。
- 詳細編集画面（EditScreen）の統合（まずはリスト表示と操作の再現に集中する）。
- 既存のネスト構造（`subItems`）への移行コードの維持（今回はフラット構造への一本化を目指す）。
- Pinia によるグローバルな状態管理の導入。
