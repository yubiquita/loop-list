# Specification: Integrate Mock-up Features (Swipe, Sort, Add, Indent)

## 1. Overview
`ref/loop-list-mock.jsx` で実装されている洗練されたUI/UX機能を、現在の Vue 3 ベースのプロジェクトに統合する。
特に、スワイプによるインデント操作、親タスク移動時の子タスク追随、およびカスケードチェックの機能を強化し、モバイルでの使い勝手を向上させる。

## 2. Functional Requirements
- **スワイプによるインデント操作 (Swipe Indent)**:
  - モバイル環境で、タスクを右スワイプするとインデントを下げ（サブタスク化）、左スワイプするとインデントを上げる（親タスク化）機能を追加。
  - スワイプ中に「サブタスクにする」「親タスクにする」という視覚的フィードバック（アイコンとテキスト）を背景に表示する。
- **スマートな並び替え (Smart Reordering)**:
  - 親タスクをドラッグした際、その配下の子タスクもひとつの「ブロック」として一緒に移動するようにドラッグ&ドロップのロジックを改善。
- **カスケードチェック (Cascade Check)**:
  - 親タスクをチェックすると、配下の子タスクがすべて自動的にチェックされる。
  - すべての子タスクがチェックされると、親タスクも自動的にチェックされる。
  - 親タスクがチェック済みの状態で、ひとつの子タスクのチェックを外すと、親タスクのチェックも自動的に外れる。
- **タスク追加 UI/UX (Add Task UI/UX)**:
  - `mock.jsx` に見られるアニメーション付きの追加フォーム、またはスムーズな入力体験を取り込む。
  - タスク入力後、キーボードの「Enter」で連続して追加できるなどの操作性を確保。

## 3. Implementation Strategy (Proposed)
- **Data Structure**: UIの実装を簡素化するため、内部データを `indent` プロパティを持つフラットなリスト形式に移行、または UI コンポーネント層でフラットに変換して扱う。
  - *理由*: `vuedraggable` による並び替え（特にブロック移動）が、再帰構造よりもフラット構造の方が制御しやすいため。
- **Components**: `ChecklistNode.vue` をスワイプ対応にアップデート。

## 4. Acceptance Criteria
- [ ] スワイプでインデントが変更され、視覚的なフィードバックが表示されること。
- [ ] 親タスクのドラッグ&ドロップ時に、その子タスクも正しく追随して移動すること。
- [ ] 親子間のチェック連動（カスケード）が正しく機能すること。
- [ ] モバイル（シミュレーター）でのタッチ操作がスムーズであること。

## 5. Out of Scope
- リストのフォルダ分けやカテゴリ管理（将来の計画）。
- バックエンド連携（引き続き LocalStorage を使用）。
