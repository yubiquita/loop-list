# Track Specification: Auto-Focus for Input Fields

## Overview
タスク追加時および新しいリスト作成時に、関連する入力欄に自動的にフォーカスを当てることで、ユーザーの入力手間を削減し、UXを向上させる。

## Functional Requirements
1.  **タスク追加時の自動フォーカス:**
    -   FAB（「新しいタスクを追加」ボタン）をクリックした際、表示されるタスク入力欄 (`.add-task-input`) に即座にフォーカスを当てる。
2.  **新規リスト作成時の自動フォーカス:**
    -   「新しいリストを作成」ボタンをクリックした際、リストセレクターを閉じずに、作成された新しいリストを編集モード (`editingListId` を設定) にし、その入力欄 (`.edit-list-input`) にフォーカスを当てる。

## Non-Functional Requirements
-   **アクセシビリティ:** スクリーンリーダーなどの支援技術においても、フォーカスの移動が適切に認識されるようにする。
-   **モバイル対応:** モバイル端末において、自動フォーカス時にソフトキーボードが表示されるようにする。

## Acceptance Criteria
-   [ ] FABをクリックすると、タスク入力欄が表示され、カーソルが当たっている。
-   [ ] モバイルでFABをクリックすると、キーボードが自動的に立ち上がる。
-   [ ] 「新しいリストを作成」をクリックすると、リスト一覧に新しい項目が追加され、その名前入力欄がフォーカス・全選択された状態になる。
-   [ ] リスト作成後、入力欄からフォーカスを外す（blur）かEnterを押すと、名前が確定される。

## Out of Scope
-   既存のリストの名前変更フローের変更（既に実装済みのため、必要に応じて共通化のみ行う）。
-   ドラッグ＆ドロップ操作時のフォーカス制御。
