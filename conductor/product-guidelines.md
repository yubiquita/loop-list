# Product Guidelines: Loop List (Vue 3 Edition)

## UX Strategy: Accessible Brave UI
「使いやすさ」と「明確さ」を最優先事項とします。
- **直感的な操作:** ボタンのサイズ、色、配置を工夫し、次に何をすべきかが一目でわかるUIを提供します。
- **フィードバック:** アクション（タスクの追加、完了、ループ）に対して、明確な視覚的または聴覚的なフィードバックを提供します。
- **高コントラストと読みやすさ:** 多様な環境で確実に情報を読み取れるよう、コントラスト比に配慮し、フォントサイズとウェイトを適切に管理します。

## Content Strategy: Concise
余計な飾りを省き、事実とアクションを簡潔に伝えます。
- **事実ベース:** ステータス表示は「3/10 完了」など、数字と事実を端的に示します。
- **能動的な指示:** 操作ラベルは「追加」「クリア」「再開」など、短く力強い動詞を使用します。
- **ノイズの最小化:** ツールチップや説明文は、本当に必要な場合にのみ最小限の文字数で提供します。

## Engineering Principles
- **KISS (Keep It Simple, Stupid):** 将来の複雑さを予見するのではなく、現在直面している問題を最もシンプルなコードで解決します。
- **Vue 3 Composition API:** リアクティビティとロジックの再利用性を高めるため、Composition API (`<script setup>`) を一貫して採用します。
- **Test-Driven Development (TDD):** 信頼性の高いアプリケーションを構築するため、機能実装の前に必ずテスト（Vitest などを使用）を記述します。
- **Component-Driven Development:** 独立性が高く、再利用可能なコンポーネントを設計することで、UIの一貫性とメンテナンス性を確保します。
